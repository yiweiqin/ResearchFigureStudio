# Critic Stage

## Summary

Use this stage after the contact sheet and editable composition are produced,
but before final delivery. The critic is a gate, not a summary. Major failures
must be fixed automatically when possible or reported as blockers.

## Required Output

Write `critic_report.md` or `critic_report.json` in the output directory.
For local `rfs` runs, also require `asset_visual_review.json` and
`asset_complexity_report.json` plus `visual_critic_iter_0.json`.

Minimum checks:

- `paper_faithfulness`: all modules, labels, formulas, and arrows trace back to
  the paper or are explicitly marked as inferred
- `slot_count`: normal system figures use at least 25 slot-level image assets
- `input_archive`: `input_manifest.json` exists and records archived paper and
  reference paths
- `layout_plan`: `layout_plan.json` exists, has at least 25 slots, and was used
  as the positioning layer before `figure_program.json`
- `image2_usage`: visual content is generated as slot assets, not one full
  architecture image
- `not_vector_only`: vector/PPT/SVG elements are structure and editable layers,
  not a replacement for requested image blocks
- `text_controllability`: critical terms, variables, formulas, metrics, panel
  IDs, and arrow labels are editable layer objects
- `ppt_editability`: `editable_composition.pptx` exists and is the main editable
  source; labels, formulas, arrows, group boxes, legends, and panel IDs are PPT
  editable objects, not baked into image2 blocks, SVG, PNG, or screenshots
- `no_semantic_crop`: no center-crop, cover-crop, forced crop, or crop-to-ratio
  that removes semantic content
- `block_integrity`: no cut-off icons, cards, characters, charts, or meaningful
  visual details
- `asset_fill_quality`: every image block has `content_fill_percent` at or above
  its `min_content_fill_percent`, pure empty margin at or below
  `max_empty_margin_percent`, and no unresolved `too_much_whitespace`,
  `tiny_centered_subject`, or `large_blank_canvas` issue
- `asset_complexity_quality`: every normal non-legend image block has enough
  visual detail to function as a research mini scene/card, not a sparse generic
  icon. Check `detail_score`, `object_count_estimate`, `simple_icon_risk`,
  `reference_crop_match`, `style_match`, and unresolved `too_simple`,
  `generic_icon`, `reference_crop_ignored`,
  `single_object_on_blank_background`, or `style_drift`
- `style_consistency`: generated blocks follow `style_sheet`
- `program_consistency`: final figure matches `figure_program.json`
- `locator_boundary`: VLM positioning, when used, outputs coordinates and arrow
  routes only; it does not write arbitrary PPT code
- `arrow_control_localization`: `reference_control_candidates.json`,
  `slot_overlay.png`, and `reference_control_overlay.png` exist; controls have
  source/target IDs, source/target anchors, multi-point normalized paths, and
  reference color tokens; final PPT renders them as editable connector shapes
- `arrow_style_quality`: `arrow_style_profile.json`,
  `selected_arrow_routes.json`, and `arrow_quality_report.json` exist; styling
  preserves reference-locked paths, improves softness/variation through PPT
  connector properties, reports crossings, bends, and obstacle overlaps, and
  uses obstacle-aware routing only for missing or explicitly fallback-allowed
  paths. If `--arrow-style-mode aesthetic` is used, every adjusted route must
  remain inside its reference tunnel, keep source-target binding unchanged, and
  render curve/bundle/halo effects as editable PPT shapes
- `reference_alignment`: when a reference image exists, major flow, group
  positions, visual density, and resource regions remain aligned unless the
  paper requires a deviation
- `asset_visual_review`: selected image blocks passed heuristic or VLM review
- `visual_critic`: reference-vs-render critic has no unresolved blocking issue

## Severity

- `major`: wrong paper mechanism, missing key module, missing PPTX source,
  PPTX that contains only one flattened full-figure screenshot, single
  full-diagram output, vector-only fallback, SVG-only delivery, semantic crop,
  baked labels, critical text generated by image2, unresolved severe whitespace
  failure, unresolved severe complexity failure such as `too_simple` or
  `reference_crop_ignored`, arrows baked into raster assets, missing control
  overlays, controls without source-target binding, arrow styling that
  overrides reference-locked paths, fallback routing applied to a reference
  locked path, aesthetic routing with `reference_tunnel_preserved=false`, or
  fewer than 25 slot assets for a normal system figure
- `medium`: inconsistent style, poor slot density, unclear arrow route, weak
  reference alignment, or incomplete prompt records
- `minor`: spacing, padding, color balance, small visual cleanup, or export
  polish issue

## Required Actions

For major failures:

- regenerate the offending image block
- regenerate or reprompt blocks whose useful content fill is below 80%
- regenerate or reprompt blocks whose pure empty margin exceeds 10%-12%
- regenerate or reprompt normal non-legend blocks that are simple standalone
  icons, ignore the local reference crop, drift from the reference style, or
  lack layered objects/micro-details
- revise `figure_program.json`
- adjust editable labels, arrows, or groups
- patch arrows only through `arrow_id`, `source_anchor`, `target_anchor`, and
  `path_percent`; do not let the critic rewrite the whole figure or emit PPT
  code
- rebuild the PPT editable layer when labels, formulas, arrows, group boxes, or
  panel IDs are baked into an image or SVG
- create `editable_composition.pptx` when it is missing
- replace vector-only placeholders with image2 slot assets
- stop delivery if image generation is unavailable and the user did not approve
  a fallback

For medium failures:

- adjust layout, style sheet, prompts, padding, or arrow routing before export
- document any intentional deviation from the reference image

For minor failures:

- fix when practical, or record as accepted residual polish issue

The final response should not claim completion if the critic report contains
unresolved major failures.
